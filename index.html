Ground Rules

📁 What lives in Drive

  • ILIA-Master-Context.md — primary AI context file
  • Living SOPs updated regularly
  • Working drafts before they become formal specs
  • Design assets, prototypes, PDFs, videos
  • Meeting notes and decision logs
  • Agent, carrier, and marketing materials

📘 What lives in Confluence

  • Formal feature specifications
  • Epics with JIRA ticket links
  • Architecture decision records (ADRs)
  • Release notes and changelogs
  • Anything needing structured comment threads
  • Confluence can link to Drive — not duplicate it

🤖 How Claude uses this Drive

  • Reads Folder 01 to maintain full ILIA platform context
  • Once GitHub arrives — extracts all user flows from routes, APIs, and DB models
  • Auto-populates Folder 02 with workflow docs from code
  • Generates Confluence-ready specs and JIRA stories
  • Acts as PM, workflow architect, and technical analyst

🚫 Non-Negotiables

  • No file lives in both Drive and Confluence
  • No adding files without a designated folder
  • Staging is temporary — never a permanent home
  • Structure approved before anything is created
  • Folder 02 expands after code review — not before
